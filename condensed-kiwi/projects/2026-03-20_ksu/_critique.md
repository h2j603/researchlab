# 비평 — KSU 프로젝트

## 핵심 주장
KSU는 CK의 산하 가상 단체로, "키위 관련 변칙 현상을 추적·경고하는 반키위 감시 단체"이며, SCP 형식의 사건 보고서 + 규칙 괴담 형식의 매뉴얼로 세계관을 구축한다. 브랜딩+웹+메뉴얼북으로 구현.

## 범위 경계

**다루는 것**:
- KSU의 세계관/컨셉 설계
- 산출물(웹, 브랜딩, 메뉴얼북) 기획
- CK 본체와의 관계 구조

**다루지 않는 것**:
- CK 자체의 브랜딩 (별도 프로젝트)
- 기술 구현 디테일 (기획 확정 후)

## 가장 약한 고리

1. **"SCP 아류"로 보일 위험.** 현재 방향(사건 보고서 + 넘버링 + [데이터 말소])은 SCP와 구조적으로 매우 유사하다. KSU만의 톤/비주얼 차별화가 없으면 "키위 버전 SCP"로 끝난다. 차별점 후보: B급 컬트의 과장된 톤, CK의 그래픽 디자인 역량, 한국어/키위 특유의 유머.

2. **미주·혁·하진의 합의가 아직 안 됐다.** 미주가 "반키위연합"을 제안했고 혁이 "재밌지만 설득력 부족"이라 했으나, 하진의 의견은 미확인. 리서치가 방향을 명확히 했지만, 팀 합의는 다음 회의에서 확인 필요.

3. **리서치 → 실제 콘텐츠로의 전환이 시작되지 않았다.** 구조와 소재는 충분히 갖춰졌다. 이제 필요한 건 리서치가 아니라 **사건 보고서 3~5개를 실제로 써보는 것**이다. 프로토타이핑 없이 리서치만 계속하면 "흥미롭지만 아무것도 안 만든" 상태가 된다.

4. **키위 팩트의 과학적 정확성.** 리서치에서 수집한 팩트(액티니딘, 알레르기 사례 등)는 모두 출처가 있지만, KSU 세계관에서 과장/왜곡할 때 어디까지가 "B급 컬트"이고 어디부터가 "잘못된 건강 정보"인지 경계가 필요하다.

## 반론

**"반키위가 맞는 방향이다"가 틀렸다면:**
- "키위를 사랑하는 사람들이 만든 키위 공포 단체"라는 이중 구조가 설명 비용을 발생시킨다. 새로 접하는 사람은 "그래서 키위를 좋아하는 거야 싫어하는 거야?"라는 혼란에 빠질 수 있다. CK의 정체성이 먼저 확립되어야 KSU의 역설이 작동한다.

**"SCP 형식이 맞다"가 틀렸다면:**
- SCP는 텍스트 기반 위키이지만, KSU의 산출물은 브랜딩+웹+메뉴얼북이다. SCP 형식은 텍스트 밀도에 의존하는데, KSU는 그래픽 디자인 팀이다. 텍스트 밀도가 아니라 **비주얼 밀도**로 세계관을 구축하는 것이 팀의 강점에 더 맞을 수 있다. 사건 보고서를 "읽는 것"이 아니라 "보는 것"으로 만들면?

## 복잡도 경고

**이전보다 나아졌지만 여전히 주의 필요.** 방향이 명확해진 건 좋지만, "SCP 형식의 사건 보고서 + 규칙 괴담 매뉴얼 + 브랜딩 + 웹"은 여전히 많다. **다음 회의에서 MVP를 정해야 한다**: 사건 보고서 3개 + 매뉴얼 1장이면 충분한가? 웹은 1페이지짜리 티저로 시작할 수 있는가?

## 실행 가능성

| 항목 | 가능 여부 | 비고 |
|------|----------|------|
| 3월말 기획 완료 | **가능해짐** | 방향 정리됨. 팀 합의만 남음 |
| 사건 보고서 프로토타입 | **가능** | 키위 팩트 확보됨. 3~5개 작성 가능 |
| 세계관 설계 | **거의 완료** | 구조(SCP+나폴리탄) 확정. 디테일만 남음 |
| 웹 구현 | **구조적 문제 유지** | 혁 군복무 중 코딩 불가, 개발자 미정 |
| 메뉴얼북 | **방향 명확** | 규칙 괴담 형식. 하진 담당 |
| 브랜딩 | **가능** | 미주 담당. "반키위" 톤 기반 |
