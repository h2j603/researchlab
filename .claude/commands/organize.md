# Organize — inbox 정리 및 리서치랩 관리

inbox에 쌓인 메모를 정리하고 적절한 위치로 분류를 제안한다.

## 실행 절차

1. `inbox/` 디렉토리의 모든 파일을 읽는다
2. 각 메모의 내용을 분석하여 다음을 제안한다:
   - **프로젝트로 발전** → `projects/`로 이동 (리서치 프로젝트로 확장할 가치가 있는 경우)
   - **관심 분야로 등록** → `areas/`로 이동 (지속적으로 추적할 주제인 경우)
   - **참고 자료로 정리** → `resources/`로 이동 (문헌 노트나 개념 정리가 필요한 경우)
   - **삭제** → 더 이상 유효하지 않은 메모
   - **유지** → 아직 숙성이 필요한 메모
3. 제안을 표 형태로 정리하여 사용자에게 보여준다
4. 사용자가 결정하면 해당 작업을 실행한다 (이동, 분류, 태그 업데이트 등)
5. 이동 시 프론트매터의 `status`와 `tags`를 적절히 업데이트한다
6. 관련 `_index.md` 파일들도 함께 업데이트한다
7. 메모들 간의 연결 관계가 보이면 함께 제안한다

## 추가 작업

- 기존 노트 중 `related` 필드가 비어있는 것이 있으면 연결을 제안한다
- 태그가 5개 이상인 주제가 있으면 MOC 생성을 제안한다

## 주의사항

- 최종 분류 결정은 반드시 사용자가 한다
- 삭제는 사용자 확인 후에만 실행한다
