# Meeting — 회의 캡처 → 회의록 + 크리틱

카카오톡/디스코드 대화 캡처(스크린샷) 또는 텍스트를 받아 회의록을 작성하고 냉철한 크리틱을 제공한다.

## 실행 절차

1. 사용자의 입력(`$ARGUMENTS`)을 확인한다:
   - 이미지 파일 경로가 있으면 → Read 도구로 스크린샷을 읽고 대화 내용 추출
   - 텍스트가 붙여넣기되었으면 → 바로 파싱
   - 아무것도 없으면 → "캡처 이미지나 대화 내용을 보내줘"라고 요청

2. 대화 내용에서 아래를 파악한다:
   - 참석자 (발언자 구분)
   - 논의 주제
   - 어떤 프로젝트에 속하는지

3. 관련 프로젝트 판단:
   - 기존 프로젝트와 연관 → `projects/YYYY-MM-DD_주제명/meetings/YYYY-MM-DD.md`
   - 불명확 → `inbox/YYYY-MM-DD_meeting-주제.md`

4. `templates/meeting-note-template.md` 형식으로 회의록 작성

5. **섹션 6(크리틱)은 반드시 작성한다** — CLAUDE.md의 "회의록 크리틱 태도" 규칙을 따른다:
   - 실제로 진전된 것 vs 말만 한 것 구분
   - 담당자·기한 없는 결정 지적
   - 비현실적 계획 경고
   - 이전 회의록이 있으면 미실행 사항 추적
   - "이 회의가 없었어도 달라지는 게 없다"면 그렇게 말한다

6. 복수 캡처 처리:
   - 여러 장이면 "더 있어?"를 확인한 후 통합
   - 한 장이면 바로 처리

7. 파일 생성 후 경로와 요약을 알려준다

## 입력 예시

- `/meeting` (이후 스크린샷 첨부)
- `/meeting 하진이랑 로고 관련 카톡` (이후 스크린샷 첨부)
- (텍스트 붙여넣기와 함께) `/meeting`

## 주의사항

- 대화 원문의 뉘앙스를 왜곡하지 않는다. 중요한 발언은 원문 발췌로 보존한다
- 크리틱은 공격적이 아니라 냉정하게. 팀원을 비난하지 않되, 문제는 정확히 짚는다
- 같은 프로젝트의 이전 회의록이 있으면 반드시 참조한다
