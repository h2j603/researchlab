# 비평 (Red Team)

## 핵심 주장
2025-2026년 크리에이티브 코딩과 웹 기술 생태계는 브라우저/모바일 접근성이 크게 향상되었으며, 아날로그 미감 시뮬레이션, 웹-투-프린트, 키네틱 타이포그래피 등 디자이너에게 유용한 도구가 다양해졌다. 특히 Paged.js(웹→인쇄), View Transitions API(페이지 전환), Scroll-Driven Animations(스크롤 기반 애니메이션) 같은 브라우저 네이티브 기술의 성숙이 "형식을 직접 설계하는 디자이너"에게 새로운 가능성을 열고 있다.

## 범위 경계
**다루는 것:**
- 웹/모바일에서 접근 가능한 크리에이티브 코딩 라이브러리
- 타이포그래피 중심 모션/인터랙션 도구
- 아날로그 미감 시뮬레이션 도구
- 아이폰 환경에서의 접근성 평가

**다루지 않는 것:**
- 각 도구의 기술적 심층 튜토리얼
- 데스크톱 전용 도구(TouchDesigner, Blender 등)의 상세 분석
- 유료 도구의 가격 비교
- 실제 작업물 제작 과정
- 주류 AI 이미지 생성 도구(Midjourney, DALL-E 등)

## 가장 약한 고리
- **실제 아이폰 테스트 부재**: "아이폰에서 접근 가능"이라는 평가가 웹 검색 정보에 기반한 것이지, 실제 아이폰에서 Hydra, OpenProcessing 등을 테스트한 결과가 아님. 터치 인터페이스에서의 코딩 경험은 데스크톱과 근본적으로 다를 수 있음
- **도구 나열 vs 실질적 유용성**: 많은 도구를 나열했지만 "그래픽 디자이너가 이것으로 무엇을 만들 수 있는가"에 대한 구체적 사례가 부족
- **가변 폰트 + GSAP 조합**: 트렌드로 소개했지만 아이폰에서 이를 실험할 구체적 방법은 불명확

## 반론
- "아이폰에서 크리에이티브 코딩"이라는 전제 자체가 비현실적일 수 있다. 작은 화면에서 코드를 작성하는 것은 탐색/학습 수준에 머물 가능성이 높고, 실제 작업물 제작은 결국 PC가 필요할 수 있음
- 도구를 많이 아는 것과 하나를 깊이 익히는 것은 다른 문제. 이 리서치가 "도구 수집"에 그치면 실제 작업에 기여하지 못함
- 아날로그 미감을 "시뮬레이션"하는 것이 진정한 의미의 아날로그 추구인지에 대한 근본적 질문이 있음

## 복잡도 경고
현재 9개 섹션, 30개 이상의 도구/기술이 나열되어 있다. **명백히 과다하다.** 이 중 실제로 손에 쥘 수 있는 것은 2-3개에 불과할 것이다.

**쳐내야 할 것:**
- 데이터 시각화 도구(섹션 7)는 현재 리서치 방향과 거리가 있음. 사회 현상을 "차트로 보여주는 것"이 너의 작업 방식이 아님
- 지식 관리 도구(섹션 9)는 이미 이 리포지토리 자체가 그 역할을 하고 있음. 도구 갈아타기는 비용만 높고 실익 적음
- Nannou(Rust 기반), React Three Fiber는 학습 비용 대비 현재 환경에서 실행 불가

**남겨야 할 것 (최대 5개):**
1. **Hydra** — Sound↔Graphic 프로젝트와 직접 연결, 아이폰에서 즉시 실험 가능
2. **Paged.js** — "형식이 세계관을 만든다"를 인쇄물에서 실현하는 유일한 경로 (PC 필요)
3. **가변 폰트 + Scroll-Driven Animations** — 키네틱 타이포 실험 (PC 필요)
4. **Cødeart** — 아이폰에서 유일하게 의미 있는 코딩 환경
5. **리소그래프 시뮬레이션** — "기술로 아날로그함을 추구"의 가장 직접적 실행

## 실행 가능성
**즉시 가능:**
- Cødeart 앱 설치하여 p5.js 기반 스케치 실험
- Hydra를 아이폰 Safari에서 열어 라이브 코딩 비주얼 테스트
- Adobe Fresco로 아날로그 브러시 실험
- Glitch Art Studio로 기존 이미지에 아날로그 텍스처 적용

**휴가 시 가능:**
- GSAP + 가변 폰트 키네틱 타이포그래피 프로토타입
- RizzCraft로 리소그래프 이펙트 작업
- c2.js 기하학 기반 제너러티브 아트 실험

**불가능 / 비현실적:**
- TouchDesigner, Nannou 등 데스크톱/설치형 도구 (군복무 중 접근 불가)
- 아이폰에서의 본격적 코딩 작업 (학습/탐색 수준에 한정될 가능성)
- 데이터 시각화 도구로 사회 현상을 "차트화"하는 접근 (작업 방향과 불일치)
- 지식 관리 도구 이전 (현재 리포지토리 + Claude 워크플로가 이미 작동 중)
