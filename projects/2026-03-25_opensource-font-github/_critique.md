# 비평 (Red Team)

## 핵심 주장
GitHub의 오픈소스 폰트 생태계는 (1) 코드 기반 자동 생성, (2) AI/ML few-shot 생성, (3) 포크/패칭이라는 세 가지 뚜렷한 패턴으로 작동하며, 각각 다른 진입 장벽과 활용도를 가진다.

## 범위 경계
- **다루는 것**: GitHub에 공개된 프로젝트의 구조, 도구, 워크플로우 패턴
- **다루지 않는 것**: 상업용 폰트 제작 파이프라인, 폰트 디자인 이론, 타이포그래피 미학 평가, Google Fonts 등록 절차

## 가장 약한 고리
- **AI/ML 프로젝트의 "실제 사용 가능성" 평가가 부족하다.** Stars 수와 학회 발표는 확인했지만, 이 코드들이 실제로 돌아가는지(의존성 충돌, 데이터셋 가용성 등)는 검증하지 않았다. README에 "works"라고 쓰여 있다고 실제로 작동하는 것은 아니다.
- **포크 프로젝트의 품질 차이를 다루지 않았다.** Nerd Fonts(62k stars)와 이름 없는 포크의 차이는 단순히 Stars 수가 아니라 유지보수 품질인데, 이를 분석하지 않았다.

## 반론
- "이 조사가 실제 폰트 제작에 도움이 되는가?"에 대한 의문. 대부분의 AI 프로젝트는 GPU 서버가 필요하고, 포크 프로젝트는 FontForge 스크립팅 능력이 필요하다. 현재 환경(아이폰 + 군복무)에서 직접 활용할 수 있는 것이 거의 없다.
- 이 리서치가 "오픈소스 폰트 생태계의 지도"인지, "내가 쓸 수 있는 도구 탐색"인지 목적이 불명확하다.

## 복잡도 경고
- 현재 규모는 적절하다. 세 축(개인 프로젝트 / AI / 포크)으로 나눈 구조는 유지할 만하다.
- 다만 각 축을 더 파고들면(예: Clova AI 모델 4개를 각각 상세 분석) 빠르게 비대해질 수 있다. 필요하면 AI 부분은 기존 `2026-03-25_ai-font-case-studies` 프로젝트로 분리하는 것이 맞다.

## 실행 가능성
- **아이폰에서 가능**: Glyphr Studio 웹 에디터 테스트, GitHub 리포 탐색 및 README 읽기
- **PC 필요**: AI/ML 프로젝트 실행(GPU 필수), FontForge 스크립팅, Iosevka 빌드 커스터마이즈
- **현실적 다음 단계**: Glyphr Studio를 아이폰 Safari에서 실제로 테스트해보는 것. Iosevka 커스텀 빌드 설정 파일을 읽어보는 것(코드를 읽는 것은 아이폰으로 가능).
